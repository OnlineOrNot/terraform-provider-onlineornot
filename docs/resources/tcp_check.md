# onlineornot_tcp_check (Resource)

Use `onlineornot_tcp_check` to verify that a TCP service accepts connections from one or more OnlineOrNot regions.

This resource uses the typed `/v1/checks/tcp` API and has TCP-specific fields instead of the generic URL check schema.

## Example Usage

```terraform
resource "onlineornot_tcp_check" "postgres" {
  name         = "Postgres TCP"
  tcp_hostname = "db.example.com"
  tcp_port     = 5432

  test_interval = 300
  timeout       = 10000
  test_regions = [
    "aws:us-east-1",
  ]
}

resource "onlineornot_tcp_check" "smtp_banner" {
  name         = "SMTP banner"
  tcp_hostname = "mail.example.com"
  tcp_port     = 25
  tcp_data     = "EHLO example.com\r\n"
}
```

## Notes

- `tcp_hostname` is the hostname or address to connect to.
- `tcp_port` must be between `1` and `65535`.
- `tcp_ip_family` defaults to `Any`.
- `tcp_data` is optional data sent after the connection opens.
- Importing a non-TCP monitor ID fails because reads go through the typed TCP endpoint.
