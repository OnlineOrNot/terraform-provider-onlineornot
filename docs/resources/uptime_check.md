# onlineornot_uptime_check (Resource)

Use `onlineornot_uptime_check` to monitor an HTTP or HTTPS endpoint from one or more OnlineOrNot regions.

This resource uses the typed `/v1/checks/uptime` API. Use it for new HTTP uptime monitors instead of the legacy generic `onlineornot_check` resource.

## Example Usage

```terraform
resource "onlineornot_uptime_check" "api" {
  name = "API health check"
  url  = "https://api.example.com/health"

  method        = "GET"
  test_interval = 60
  timeout       = 10000
  test_regions = [
    "aws:us-east-1",
    "aws:eu-central-1",
  ]

  assertions = [
    {
      type       = "TEXT_BODY"
      property   = ""
      comparison = "CONTAINS"
      expected   = "ok"
    }
  ]
}
```

## Notes

- `url` is required for URL-based uptime checks.
- `type` is managed by the provider and is always `UPTIME_CHECK` for this resource.
- Importing a non-uptime monitor ID fails because reads go through the typed uptime endpoint.
