# onlineornot_dns_check (Resource)

Use `onlineornot_dns_check` to verify that a DNS name resolves as expected from one or more OnlineOrNot regions.

This resource uses the typed `/v1/checks/dns` API and has DNS-specific fields instead of the generic URL check schema.

## Example Usage

```terraform
resource "onlineornot_dns_check" "root_a" {
  name            = "example.com A record"
  dns_domain      = "example.com"
  dns_record_type = "A"
  dns_protocol    = "UDP"

  test_interval = 300
  test_regions = [
    "aws:us-east-1",
    "aws:eu-west-2",
  ]
}

resource "onlineornot_dns_check" "custom_resolver" {
  name            = "example.com via Cloudflare DNS"
  dns_domain      = "example.com"
  dns_record_type = "A"
  dns_resolver    = "1.1.1.1"
  dns_protocol    = "UDP"
}
```

## Notes

- `dns_domain` is the domain name to query.
- `dns_record_type` is the record type to query, such as `A`, `AAAA`, `CNAME`, `MX`, `NS`, `SOA`, or `TXT`.
- `dns_resolver` is optional. Leave it unset to use the platform default resolver.
- `dns_protocol` defaults to `UDP`.
- Importing a non-DNS monitor ID fails because reads go through the typed DNS endpoint.
